本地ip地址的获取（获取ens33网卡）

ip -f inet a show ens33| grep inet| awk '{ print $2}' | cut -d/ -f1

其中centos7开始，网卡名字叫ens33，如果网卡是其它名字，就修改ens33。

  • 第一段 ip -f inet a show ens33是获取本地网卡的ip地址，其中-f inet是获取ipv4的地址。
  • grep inet是获取ip哪一行。
  • awk '{ print $2}'打印第二个字段，命令中为ip字段。
  • cut -d/ -f1是去掉了ip中最后的网段。

vim中的设置

vim的用户配置文件应该是在/usr/bin/vim/.vimrc中，但是本人配置了.vimrc后不起作用，所以对系统的vimrc文件/etc/vimrc进行修改。

syntax enable “自动检测语法
syntax on “自动语法高亮
set number “显示行号
set tabstop=4 “表示Tab代表4个空格的宽度
set expandtab “表示Tab自动转换成空格
set autoindent “表示换行后自动缩进
set history=400 “vim记住的历史操作的数量，默认的是20
set smartindent “智能对齐
set shiftwidth=4 “偏移4个空格
set cursorline “每行都有下划线

不用箭头键进行上下左右移动光标

h是左移，l是右移，j是下移，k是上移

上下翻页

pageup 上翻页，pagedown 下翻页

进入输入模式

a 光标后插入，i 光标前插入， o 下一行插入

删除

单行删除 dd，全文删除 dG

复制

单行 在命令模式下，将光标移动到将要复制的行处，按“yy”进行复制，将光标移动到将要粘贴的行处，按“p”进行粘贴。

多行 在命令模式下，按v进入可视模式，按箭头键即可选择相应的行，选择相应的行之后，到复制的文本结束的地方，再按y进行复制，之后会自动退出退出可视模式，光标返回到文本开始的位置。移动光标到需要开始复制的地方，按p进行复制

linux文件权限

Linux的文件权限有九种，分别是拥有者（owner）、所属群组（group）、其他人（others）三种身份各有自己的读（read）、写（write）、执行（execute），文件的权限字符为：[-rwxrwxrwx]，这九个权限是三个三个一组。其中，我们可以使用数字来代表各个权限，各权限的数字对照表：

r:4 w:2 x:1

使用chmod来改变文件的权限（需要sudo或者root权限），如chmod 777 file就是将文件的权限修改为[-rwxrwxrwx]（都是可读可写可执行），而chmod 765 filename就是将文件权限修改为[-rwxrw-r-x]（拥有者可读可写可执行，所属群组可读可写，其他人可读可执行）。

当然也可以不用数字来进行文件权限的修改，可以使用chmod u+x file的方法给file文件的所有者增加可执行权限。

其中u代表用户，u可以替换为a（代表所有）、g（代表用户组）、o（代表其他），x代表执行的权限，可以替换为r（可执行），w（可写）。+ 表示增加权限。

这意味着chmod u+x somefile 只授予这个文件的所属者执行的权限
而 chmod +x somefile 和 chmod a+x somefile是一样的

跟着尚硅谷的k8s教程在学kubernetes，期间也遇到了一些问题，分享一下这些问题。

node加入时没有反应，报couldn’t validate the identity of the API Server

Token过期了，需要重新生成token和sha256。

生成Token的方法。 kubeadm token create，在master服务器输入命令可以生成相应的Token。

再生成使用证书生成Token相应的sha256。命令openssl x509 -pubkey -in /etc/kubernetes/pki/ca.crt | openssl rsa -pubin -outform der 2>/dev/null | openssl dgst -sha256 -hex | sed 's/^.\* //'

最后再join一下就完事了。kubeadm join 对应的ip和端口 --token 对应的token --discovery-token-ca-cert-hash sha256:对应的sha256

centos内核更新没反应

首先通过uname -r的命令来查看os内核版本。

然后升级内核

导入ELRepo仓库的公共密钥

安装ELRepo仓库的yum源

安装最新版本内核

设置grub2

内核安装好后，需要设置为默认启动选项并重启后才会生效。

查看系统上的所有可用内核：

设置新的内核为grub2的默认版本

服务器上存在4 个内核，我们要使用 4.18 这个版本，可以通过 grub2-set-default 0 命令或编辑 /etc/default/grub 文件来设置

重启就完事了

修改完网卡配置后还ping不通

不要忘了重启，重启命令是reboot。

windows中hosts无法修改

添加了权限都无法修改，研究了半天，发现是没有取消文件的只读权限。右键文件，只读的勾取消掉就行了。

harbor突然打不开了

原来是有关harbor的一些容器down了，需要重新run一下

kubectl run之后状态一直ready 0/1

使用kubectl describe pod查看pod信息后，发现出现了CrashLoopBackOff的问题，仔细一想后发现，容器hello-world不是一个可持续化的容器，自己写了一个静态化的容器后再run就没什么问题了。

在k8s中，一般使用yaml格式的文件来创建符合我们预期期望的pod，这样的yaml文件我们一般称为资源清单

名称空间级别

消息队列

发表于 2020-07-23 更新于 2020-07-24 分类于 Middle Layer 评论数：

消息队列是分布式系统中重要的组件，在很多生产环境如商品抢购等需要控制并发量的场景下都需要用到。

消息队列(MQ)概述

消息队列（Message Queue），是分布式系统中重要的组件，其通用的使用场景可以简单地描述为：当不需要立即获得结果，但是并发量又需要进行控制的时候，差不多就是需要使用消息队列的时候。

消息队列主要解决了应用耦合、异步处理、流量削锋等问题。

当前使用较多的消息队列有RabbitMQ、RocketMQ、ActiveMQ、Kafka、ZeroMQ、MetaMq等，而部分数据库如Redis、Mysql以及phxsql也可实现消息队列的功能。
