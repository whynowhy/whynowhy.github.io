使用python生成excel或csv文件遇上的问题

发表于 2019-07-20 更新于 2019-08-03 分类于 Python

使用pandas.DataFrame.to_excel()写入。

假设字典dict={K_A:V_A, K_B:V_B, K_C:V_C}要生成表格

使用命令pandas.DataFrame(dict).to_excel('磁盘')进行生成，如果V_A只是一个值，不能迭代，会报错： ValueError: If using all scalar values, you must pass an index。这是因为 pandas 的 DataFrame 方法需要传入一个可迭代的对象(列表，元组，字典等)，DataFrame 指定 index 参数就可以解决这个问题。修改后为命令为pandas.DataFrame(dict,index=[0]).to_excel('本地地址')。生成的excel表格第一列会显示index,在to_excel()方法中加入index=False方法即可去除。生成的单行数据表格的形式如下：

K_A K_B K_C
V_A V_B V_C

如果需要生成多行数据，那么字典中的键值对的值必须是列表，如字典dict={K_A:[V_A1,V_A2], K_B:[V_B1,V_B2], K_C:[V_C1,V_C2]}可生成以下表格：

K_A K_B K_C
V_A1 V_B1 V_C1
V_A2 V_B2 V_C2

列表元素不一致时，报arrays must all be same length错

如果字典中键值对中的value中的列表的元素数量不一样，如dict={K_A:[V_A1,V_A2], K_B:[V_B1,V_B2], K_C:[V_C1]}，就无法使用pandas.DataFrame().to_excel()生成表格，会报错：ValueError: arrays must all be same length。原因是因为使用dataframe(dict) 构建dataframe时，每个key会变成一个column，list-like values会变为行，每个values中的list长度不一致就会出现这个错误。如果需要使用to_excel生成表格，就需要将不一样长度的列表填充空值，或者使用xlsxwriter方法来生成excel，相关代码如下：

数据库的相关知识

发表于 2019-07-20 更新于 2019-08-28 分类于 Database

关系型数据库

平常使用的关系型数据库（如mysql）中，主要分为两种类型，分别是innodb和myisam。其中innodb和myisam主要的差别为：

  1. 事务和外键：InnoDB具有事务，支持4个事务隔离级别，回滚，崩溃修复能力和多版本并发的事务安全，包括ACID。如果应用中需要执行大量的INSERT或UPDATE操作，则应该使用InnoDB，这样可以提高多用户并发操作的性能。MyISAM管理非事务表。它提供高速存储和检索，以及全文搜索能力。如果应用中需要执行大量的SELECT查询，那么MyISAM是更好的选择
  2. 全文索引：早期innodb不支持全文索引，在mysql5.6版本之后，innodb开始支持全文索引，5.7版本之后，通过使用ngram支持中文。
  3. 锁：MyISAM支持表级锁定，提供与 Oracle 类型一致的不加锁读取(non-locking read in SELECTs)。InnoDB支持行级锁，InnoDB表的行锁也不是绝对的，如果在执行一个SQL语句时MySQL不能确定要扫描的范围，InnoDB表同样会锁全表，注意间隙锁的影响
  4. 存储：MyISAM在磁盘上存储成三个文件。第一个文件的名字以表的名字开始，扩展名指出文件类型，.frm文件存储表定义，数据文件的扩展名为.MYD，索引文件的扩展名是.MYI。InnoDB，基于磁盘的资源是InnoDB表空间数据文件和它的日志文件，InnoDB 表的大小只受限于操作系统文件的大小
  5. 索引：InnoDB使用的聚簇索引、索引就是数据，顺序存储，因此能缓存索引，也能缓存数据。MyISAM使用的是非聚簇索引、索引和文件分开，随机存储，只能缓存索引。
  6. 并发：MyISAM读写互相阻塞：不仅会在写入的时候阻塞读取，MyISAM还会在读取的时候阻塞写入，但读本身并不会阻塞另外的读。InnoDB读写阻塞与事务隔离级别相关。

读书笔记--《Incremental Ensemble Classifier Addressing Non-stationary Fast Data Streams》

发表于 2019-05-14 更新于 2019-07-28 分类于 学习

主题摘要

本文为了处理不平稳的数据流（概念漂移数据流），本文提出了一种名为Modal Mixture Model的方法，简称为3M,该方法使用集成的基分类器进行分类，并在在线方法中使用强化学习的方法来调整集合中每个基分类器的权重。当初始分类器被搭建完成通过检验并置入数据流后，分类器随机选择数据流中的实例作为新训练数据，然后根据所得到的训练精度来更新基分类器的权重。

详细算法

  1. 集成算法的结构：该集成算法包含一组VFDT基分类器和朴素贝叶斯分类器，使用weighted bagging来使算法变得多元化。当训练数据进入系统时，基于该分类器的当前准确度加权，为每个成员分类器分配不同的权重。

  2. 成员的多样性：VFDT和朴素贝叶斯分类器都可以进行增量训练，且内存的执行效率很高。且同时可以处理数字数据和定类数据。集成基分类器的多样化是集成算法高鲁棒性的关键。该集成算法多样化来自算法选择，因为朴素贝叶斯算法被认为是生成算法而VFDT是一种判别方法，训练分类器时的bagging和wagging方式进一步使成员多样化。

  3. 详细的算法:与普通的集成算法有两点不同，加权和多样化的VFDT 和朴素贝叶斯基学习器的异构集合，以及基于Bellman方程的方法用于在线更新集合权重与基分类器的复位。

    1. 算法的预测方法：和普通的动态权值方法一致

    2. 算法的训练方法：如下方的算法图，4-6行进行权重更新，8-9行是重置成员分类器的条件测试,9-11行是对集合成员分类器执行重置过程。

    3. 各个基分类器权重的更新，预测正确时权重w=(1-λ)+λw，预测错误时w=λw，其中λ是learning factor,0.5<λ<1。

    4. 当w<0.05时，基分类器就需要被重置，重置时的训练集为缓存中保存的最后50个实例。

读书笔记--《Online Active Learning Ensemble Framework for Drifted Data Streams》

发表于 2019-04-29 更新于 2019-07-28 分类于 学习

主题摘要

该篇论文的的中文标题为《应用于概念漂移数据流的在线主动集成学习框架》，这篇论文在混合标记策略的基础上提出了一种新的应用于漂移数据流的主动学习框架。将一个集成分类器划分为一个长期稳定的分类器以及多个动态分类器，动态分类器是使用多级滑动窗口来创建和更新从而来处理渐进或者突发的数据流漂移。本文在学习方式上使用了主动学习的策略，采用不确定性和随机策略来标记实例。不确定策略的决定阈值是动态调整的，当发生概念漂移时，阈值会相应降低，来查询最不确定的实例。

在线主动学习框架

该论文将数据流划分为多个（k个）子数据流，使用多级联滑动窗口的方法来更新集成分类器中弱分类器，其中一个子分类器（长期稳定分类器）使用一条子数据流进行长期更新，而其它的子数据流来生成动态分类器。当子数据流生成k-1个动态分类器后，不使用在线更新的方法对动态分类器进行更新而是直接使用子数据流生成新的动态分类器。

主动在线学习策略

在数据流分类时，计算实例A分类结果的准确性，如果准确性小于阈值，看是否满足不确定策略（Uncertainty Strategy）和随机策略（Random Strategy），如果满足使用人工主动学习的方式标记实例，从而更新稳定分类器和最后一个动态分类器。
