一月新番回顾和四月新番导视

发表于 2019-04-02 更新于 2019-07-28 分类于 随笔

一月新番回顾

本来今年一月就想做一次季番的回顾与导视了，结果因为要写国家自然科学基金搁置了，其实这次的新番导视我也有点想咕咕咕的，但是既然之前决定了，那就做一次。

首先回顾一下一月我看的新番，有灵能第二季，辉夜大小姐想让我告白，马纳利亚的魔法学院，约定的梦幻岛，多罗罗，荒野的寿飞行队，不吉波普不笑和加油女友吧，哦，还有看了两集就弃了的五等分的花架，差不多7到8部吧，季度追番的平均水平吧，不过因为最近看动画片的兴致越来越低了，导致经常看到一半就鸽了。

接下来一部一部的分析吧。

灵能百分百 第二季

骨头社制作加上one的原作，可以说是不负众望，不管是整部作品的剧情还是作画都可以说是炸裂，bangumi上也给出了8.5分超高分。

首先灵能百分百作为一部漫改作品，one老师的原作就已经给整部作品打下了坚实的基础，虽然原作画的比较抽象吧，但是漫画正真厉害的在于分镜，原作的漫画直接可以当动画分镜来用了。再吹一波骨头社吧，可以说是真正的用心在做动画，践行自己公司要做有骨气的动画的建设期望。除了骨头社自己最擅长的打戏，整部作品的文戏也非常的深入人心，比如说其中的第7集，将灵幻这个人的内心剖析的非常精准，非常的有代入感。再说说动作戏，这时候就要单独把11集拿出来品一品了，整集中，不管是甩技能还是近身格斗，所有的动作都是行云流水，再配合上骨头社所特有的中村丰方块式的爆炸，只能说一个字，爽，可以说这一整集都能上anitamshi的作画mad。

总体上而言，整部作品没有太大的瑕疵，非常推荐，个人推荐指数10颗星。

辉夜大小姐想让我告白

kaguyasama这部作品绝对是这个季度最让人惊喜的作品了，本来以为只是一部烂大街的漫改校园恋爱喜剧，但是万万没想到，制作组竟然有这么高的厨力，毕竟这年头不毁原作已经在我心中成为高标准了。

其实整部动画的故事很简单，就是女主辉夜和男主白银互相斗智斗勇（发狗粮）的作品，话说在原作中本来还是在月刊连载时还有一种互相智斗的感觉，到了周刊连载后只剩下无尽的发狗粮了。除了男女主角外，还有一个搅屎棍子千花书记，本来以为只是一个工具人，但是就是因为第三集的ed，完全手绘90s跳舞，还只用了一个ED，真的是奢侈啊，第三集播出之后感觉全世界都在跳书记舞（我也就看了100遍而已）。整部作品的完成度可以说是非常的高，人物形象，动作细节，肢体语言都表现的非常好，制作组可以说是非常的上心了。

个人还是比较喜欢这种恋爱喜剧的，推荐指数9.5颗星。

马纳利亚的魔法学院

整部作品就充斥着一个单词“AWSL”，虽然我不玩影之诗，不了解整个故事的背景，但是我就是想看安和古蕾雅搞姬。

这部作品就讲了安和古蕾雅在马纳利亚学院中学习生活的日常生活，轻松悠闲。外加制作方是Cygames，钱多任性，画面相当精美。

百合控必看，推荐指数9颗星。

不吉波普不笑

不吉波普不笑这部作品已经有了20年的历史了，物语系列和空之境界都号称受过不吉波普的启发，外加制作方是MAD HOUSE，再加上令人惊艳的PV，所以我对这部作品还是非常期待的。

整部作品的叙事方法并没有按照时间的顺序来进行推进，而是跟着人物来进行推进，简而言之就是作品一场事件中每个角色眼中所遇到的情况给叙述了一遍，而不是用上帝视角来看，只有当所有角色的故事都叙述完成之后，你才能了解这个故事，而且不同的事件之间也有着联系。所以看完第一集，整个人都是懵的，有一种我是谁，我在哪，我在看什么的感觉，看完第三集，也就是第一个故事结束，就大概了解了个大概，当我把18集，5个作品看完，就知晓了作品的前因后果。这样的叙述作品的方式，虽然能够更加完整叙述作品的主旨，但是对于一些不用心看了来说简直就是灾难。这部作品的制作班底虽然很雄厚吧，但是有一些角色的人设实在是不咋地，第一个故事中动作戏也是有一点敷衍。

这部作品推荐给那些能够沉得下心来的人去观看，推荐指数8颗星。

荒野的寿飞行队

水岛努加横手美智子的超强组合，外加看完PV之后给人以一种少女与战车的感觉，给我带来了非常高的期待，但是整部作品看完之后还是有一点稍稍的失望的。

整部作品是架空在现实之上的，作品中的国家都被大山所分开，所以交通工具只能是飞机，战争啊，抢劫啊，都只能靠飞机，然后主角团们作为护卫队和其它势力战斗的场面成为了这部作品最主要的看点。当然寿飞行队这部作品在机站场面上没有让我们失望，我虽然对狗斗啊，飞机啊都不怎么了解，但是就是看的很爽，由于全篇使用的都是3D作画，整个空战的场面的镜头都无比顺畅。不过除了空战，好像也没什么好吹的了，尤其是到最后又来了一个空中之门的设定，这个门是与其它的世界相连的，坏蛋想夺走这个门的占有权，而主角团最后却把门给炸了，让我最后都不知道门的后面到底有什么，感觉故事没讲完一样。

这部作品推荐给那些喜欢飞机和喜欢大型机械战斗场面的人去观看，推荐指数8颗星。

家有女友

我是一季度动画都结束后看了gigguk介绍才去看的这部作品，可以说是一部很差劲却很对我胃口的家庭伦理大作。

该作品讲述了一个男主发现自己的父亲的再婚对象的两个女儿一个是自己喜欢的老师，而另一个是自己一夜情的对象，然后三人在一个屋檐下一起生活，骨科的故事。这部作品虽然作画真的很差劲，阳菜的脸有时候在一集中有多个样子，然后时不时放出长达5秒左右的静止帧，如果是一部普通的作品，那绝对会把我给劝退了。但是这部作品实在是太刺激了，很符合SPICY这个词好吧，每次看完一集总让人欲罢不能的想看下一集，看看男主又能做出怎样的操作，看看女主们还能玩出什么我想不到的套路来。

这部作品推荐那些喜欢吃胃药的同学去看，个人推荐指数8颗星。

约定的梦幻岛

话说这部约定的梦幻岛在还没动画化的时候就被称为台柱岛，JUMP的希望，可以说是非常吸引人眼球吧。

于是我看到网上吹得那么厉害，就去DMZJ看了一下，发现是真的好看啊，尤其是漫画的第一部分，脱狱篇。整个故事的连贯性很强，漫画的画风也特备好，一页一页的翻下去，故事的带入性特别强，看得特别紧张。就是因为这个漫画，害的我的开题答辩挂了。但是我在看动画的时候就没有那种紧张感，可能是因为我都知道后续的发展，所以我在第三集就弃了。

其实讲道理这部作品还是不错的，cl的制作也还行，推荐指数7.5颗星。

多罗罗

手冢治虫老师的原作，MAPPA的制作。

这部作品讲了一个被抛弃的男孩，斩杀各种恶魔，找回自己身体的故事。说实话，这部作品看来很不错，但是不知道为什么，就是没有看下去的动力。

我就不评分了。

四月新番导师

这次四月被称为最弱四月，看起来不错的作品基本都是续作，比如一拳超人，进击的巨人等。还有一些大牌导演的电波作，像卡罗拉和缇丝黛，皿三昧这些。我反正看了一下列表，就准备追这些作品：CAROLE & TUESDAY，皿三昧，进击的巨人 第三季 Part.2，鬼灭之刃，异世界四重奏和一拳超人。

码字好累啊，估计这是我第一次也是最后一次做新番点评了。

Spring框架简介

Spring是一个轻量级控制反转(IoC)和面向切面(AOP)的容器框架。

Spring框架的核心功能有两个

  • Spring容器作为超级大工厂，负责创建、管理所有的Java对象，这些Java对象被称为Bean。
  • Spring容器管理容器中Bean之间的依赖关系，Spring使用一种被称为”依赖注入”的方式来管理Bean之间的依赖关系。

JavaBean的简介

JavaBean 是一种JAVA语言写成的可重用组件。为写成JavaBean，类必须是具体的和公共的，并且具有无参数的构造器。JavaBean 通过提供符合一致性设计模式的公共方法将内部域暴露成员属性，set和get方法获取。

JavaBean的组成

(1) 属性 (2) 方法 (3) 事件

JavaBean的任务目标

JavaBean的任务就是: “Write once, run anywhere, reuse everywhere”，即“一次性编写，任何地方执行，任何地方重用”。这个任何实际上就是要解决困扰软件工业的日益增加的复杂性，提供一个简单的、紧凑的和优秀的问题解决方案。

Spring容器中的Bean

指定容器中Bean的作用域

当通过Spring容器创建一个Bean实例时，不仅可以完成Bean实例的实例化，还可以为Bean指定特定的作用域。如果不指定Bean的作用域，Spring默认使用singleton作用域。prototype作用域的Bean的创建、销毁代价比较大。而singleton作用域的Bean实例一旦创建成果，就可以重复使用。因此，应该尽量避免将Bean设置成prototype作用域。

使用自动装配注入合作者Bean

Spring能自动装配Bean与Bean之间的依赖关系，即无须使用ref显式指定依赖Bean，而是由Spring容器检查XML配置文件内容，根据某种规则，为调用者Bean注入被依赖的Bean。

Spring中Bean的3中创建方法

(1) 使用构造器创建Bean实例 (2) 使用静态工厂方法创建Bean (3) 调用实例工厂方法创建Bean

Spring中的依赖注入

使用依赖注入，不仅可以为Bean注入普通的属性值，还可以注入其他Bean的引用。依赖注入是一种优秀的解耦方式，其可以让Bean以配置文件组织在一起，而不是以硬编码的方式耦合在一起。

使用Spring框架之后，调用者无需主动获取被依赖对象，调用者只要被动接受Spring容器为调用者的成员变量赋值即可，由此可见，使用Spring后，调用者获取被依赖对象的方式由原来的主动获取，变成了被动接受——所以Rod Johnson称之为控制反转。

另外从Spring容器的角度来看，Spring容器负责将被依赖对象赋值给调用者的成员变量——相当于为调用者注入它依赖的实例，因此Martine Fowler称之为依赖注入。

Spring中的AOP

什么是AOP

AOP（Aspect Orient Programming）也就是面向切面编程，作为面向对象编程的一种补充，已经成为一种比较成熟的编程方式。其实AOP问世的时间并不太长，AOP和OOP互为补充，面向切面编程将程序运行过程分解成各个切面。

AOP专门用于处理系统中分布于各个模块（不同方法）中的交叉关注点的问题，在JavaEE应用中，常常通过AOP来处理一些具有横切性质的系统级服务，如事务管理、安全检查、缓存、对象池管理等，AOP已经成为一种非常常用的解决方案。

使用AspectJ实现AOP

AspectJ是一个基于Java语言的AOP框架，提供了强大的AOP功能，其他很多AOP框架都借鉴或采纳其中的一些思想。其主要包括两个部分：一个部分定义了如何表达、定义AOP编程中的语法规范，通过这套语法规范，可以方便地用AOP来解决Java语言中存在的交叉关注点的问题；另一个部分是工具部分，包括编译、调试工具等。

maven的简介

发表于 2019-03-25 更新于 2019-07-28 分类于 Java

maven是什么

Maven 翻译为”专家”、”内行”，是 Apache 下的一个纯 Java 开发的开源项目。基于项目对象模型（缩写：POM）概念，Maven利用一个中央信息片断能管理一个项目的构建、报告和文档等步骤。

Maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理

配置目录

maven 提倡使用一个共同的标准目录结构，Maven 使用约定优于配置的原则，大家尽可能的遵守这样的目录结构。
目录结构详见http://www.runoob.com/maven/maven-tutorial.html

maven pom

POM( Project Object Model，项目对象模型 ) 是 Maven 工程的基本工作单元，是一个XML文件，包含了项目的基本信息，用于描述项目如何构建，声明项目依赖，等等。

执行任务或目标时，Maven 会在当前目录中查找 POM。它读取 POM，获取所需的配置信息，然后执行目标。

maven构建的生命周期

Maven 构建生命周期定义了一个项目构建跟发布的过程。

一个典型的 Maven 构建（build）生命周期是由以下几个阶段的序列组成的：开始 → Validate(验证项目) → compile(执行编译) → test(测试) → package(打包) → verify(检查) → install(安装) → deploy(部署)

博客的创建

在命令行中进入博客的文件夹， 使用hexo new "博客的名字"创建一篇博客。

博客的编写

创建之后博客的md文件存储在/blog/source/_post中，编辑相关md文件即可编写博客。注：tag的形式为[tag1,tag2,tag3……]。

未发布博客的查看

使用hexo server即可启动本地服务器，通过输入http://localhost:4000即可进入本地的博客浏览编写的博客。

发布博客

使用hexo deploy -git即可发布博客。

npm config set registry https://registry.npm.taobao.org
之后输入一下内容进行测试：
npm config get registry

我引用的博客中写的是:
ssh-keygen -t- rsa -C "your email"
其实应该是：
ssh-keygen -t rsa -C "your email"
